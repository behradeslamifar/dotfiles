# Codex Memory Bank

Use this folder to keep durable project context for Codex runs.

## Files
- `projectbrief.md`: project purpose, scope, and constraints
- `productContext.md`: user needs and product goals
- `activeContext.md`: current focus, recent decisions, and open questions
- `systemPatterns.md`: architecture, conventions, and patterns
- `techContext.md`: stack, tooling, and environment notes
- `progress.md`: milestones, changelog-style progress, and next steps

Update these files regularly so future Codex sessions have accurate context.
